Multiplayer


My project uses custom replication, as the standard Unreal "character" doesn't fit. On joining or hosting a game, some basic game and pawn setup is done in the Game Mode as covered in Game Mode Flow. Pawns go through a BeginPlay setup for all clients and a secondary OnPossessed setup for pawns belonging to the local player. Remotely owned pawns have physics disabled, raycast suspensions report depth values for animation but do not run their force or friction calculations to prevent duplicate work. Local pawns retain their physics functionality and broadcast their transforms and event actions to the server, where they are multicast or auto-replicated to all other players. Most pawn actions are client-auth as the project's main goal is an enjoyable PVE experience. Gunfire is run from a timeline to prevent RPC overflow, one reliable RPC starts the firing sequence, and another ends it. Gunfire patterns are drawn from a random stream, ensuring identical "random" fire patterns on both owner and remote pawns, with only the interpolated pawn transforms differing.
